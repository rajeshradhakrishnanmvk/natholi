# Background Tasks Implementation Summary

## Overview

The Background Tasks feature allows users to run long-running bot tasks asynchronously while continuing to use the chat interface. This document outlines the technical implementation.

## Architecture

### Core Components

1. **BackgroundTaskManager Class** (`index.html`)
   - Manages task lifecycle (create, execute, cancel, delete)
   - Maintains active and completed task queues
   - Handles persistence via localStorage
   - Renders task UI dynamically

2. **Task Queue System**
   - `activeTasks`: Map of currently running tasks
   - `completedTasks`: Map of finished tasks (max 20)
   - Tasks stored with unique IDs generated by `generateId()`

3. **UI Components**
   - Background Task Modal (creation)
   - Tasks Panel (monitoring)
   - Notification System (alerts)
   - Badge Counter (active task count)

## Data Structure

### Task Object
```javascript
{
    id: "unique_task_id",
    botId: "bot_123",
    botName: "Creative Writer",
    botModel: "gemma4:e4b",
    prompt: "Write a story about...",
    taskName: "Short Story",
    status: "running|completed|failed|cancelled",
    startedAt: 1234567890,
    completedAt: 1234567890,
    duration: 5000,
    result: "The generated content...",
    error: null
}
```

## Key Functions

### BackgroundTaskManager Methods

#### `startTask(botId, prompt, taskName)`
- Creates new task object
- Adds to activeTasks map
- Triggers async execution
- Shows notification
- Updates UI

#### `executeTask(task)`
- Builds message history
- Calls Ollama API with `stream: false`
- Handles success/failure
- Moves task to completed
- Shows completion notification

#### `cancelTask(taskId)`
- Marks task as cancelled
- Moves to completed tasks
- Updates UI

#### `viewResult(taskId)`
- Retrieves completed task
- Inserts result into current chat
- Adds to conversation history

#### `saveToStorage()` / `loadFromStorage()`
- Persists tasks to localStorage
- Restores completed tasks on reload
- Does not restore active tasks (they're no longer running)

### Background Task Modal
- Task name input (optional)
- Bot selector (populated from available bots)
- Prompt textarea
- Create/Cancel buttons

### Tasks Panel
- Slides in from right
- Shows active tasks (with cancel button)
- Shows completed tasks (with view/delete buttons)
- Clear all button for completed tasks

### Notification System
- Fixed position top-right
- Auto-dismiss after 5 seconds
- Color-coded by type (info, success, error, warning)
- Slide-in animation

## Integration Points

### Bot System Integration
1. **Bot Selection**: `populateBackgroundTaskBots()` populates dropdown
2. **Bot History**: Tasks use bot's systemPrompt if available
3. **Result Display**: Results rendered with markdown parsing

### Chat Integration
1. **Non-blocking**: Tasks run independently of main chat
2. **Result Insertion**: `viewResult()` adds to current conversation
3. **Metadata**: Results tagged with `isBackgroundTask: true`

### Storage Integration
1. **localStorage Key**: `'background_tasks'`
2. **Data Format**: 
   ```json
   {
     "active": [],
     "completed": [...]
   }
   ```
3. **Limit**: 20 most recent completed tasks

## Execution Flow

### Starting a Task
```
User clicks "Start Task"
  ↓
Modal opens with form
  ↓
User fills: bot, prompt, name
  ↓
createBackgroundTask() called
  ↓
BackgroundTaskManager.startTask()
  ↓
Task added to activeTasks
  ↓
executeTask() runs async
  ↓
UI updates, notification shown
```

### Task Completion
```
Ollama API returns response
  ↓
executeTask() receives result
  ↓
Task moved to completedTasks
  ↓
UI updated (panel, badge)
  ↓
Completion notification shown
  ↓
State saved to localStorage
```

### Viewing Results
```
User opens Tasks Panel
  ↓
Finds completed task
  ↓
Clicks "View" button
  ↓
viewResult(taskId) called
  ↓
Result inserted into chat
  ↓
Added to conversation history
  ↓
Scrolls to show result
```

## Error Handling

1. **Network Errors**: Caught in executeTask, task marked as failed
2. **Empty Prompt**: Validation in createBackgroundTask()
3. **No Bots**: Bot selector checks for available bots
4. **Ollama Offline**: Graceful failure with error notification

## Performance Considerations

1. **Async Execution**: Tasks don't block UI thread
2. **Task Limit**: Max 20 completed tasks in storage
3. **Memory Management**: Completed tasks can be cleared
4. **No Polling**: Event-driven updates, no unnecessary refreshes
